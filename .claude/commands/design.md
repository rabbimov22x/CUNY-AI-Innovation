# /design

Use this command when improving the visual design, UX, content structure, or UI polish of the StartNow product.

## Goal

Make the interface feel credible, modern, and motivating for both CUNY students and local small businesses while preserving product clarity and conversion.

## Product Lens

This is not a generic freelancer marketplace. It is an equity-driven platform that helps verified CUNY students access paid contract work, build real portfolios, and create a record of professional experience before graduation.

Design decisions should reinforce:

- trust
- legitimacy
- upward mobility
- simplicity
- momentum toward paid work

## Instructions

When asked to design or redesign a screen:

1. Identify the user type:
   - student
   - employer
   - shared workflow
2. Clarify the screen's primary action.
3. Prioritize usability before decoration.
4. Keep important trust signals visible:
   - verification
   - ratings
   - project status
   - budget or payment details
5. Write product copy that is specific, calm, and credible.
6. Use existing Next.js, Tailwind, and component patterns in the repo.
7. Make the result responsive for mobile and desktop.

## Visual Direction

Aim for:

- strong hierarchy
- clean layouts
- clear calls to action
- modern but grounded styling
- purposeful use of color

Avoid:

- generic SaaS landing-page aesthetics
- empty marketing fluff
- cluttered dashboards
- visual choices that make the platform feel childish or untrustworthy

## Output Expectations

Unless the user asks otherwise:

- implement the design directly in code
- keep components readable and reusable
- explain the main UX changes briefly
- note any assumptions made
